Why GitMap

Every GIS team has these problems.

GitMap solves the version control gap that's been missing from ArcGIS workflows since the beginning.

"Who changed the basemap last Tuesday?"
Full commit history with author, timestamp, and message.
gitmap log
"I need to test this symbology without breaking production."
Isolated branches — experiment freely, merge when ready.
gitmap branch feature/symbology
"Revert the map to last week's state."
Checkout any previous commit instantly.
gitmap checkout <commit-id>
"What's different between staging and production?"
Layer-by-layer visual diff between any two branches.
gitmap diff --branch production
"Keep 50 maps in sync with Portal changes."
Auto-pull tracks remote map state and syncs on schedule.
gitmap auto-pull
"Share map state between team members."
Push/pull to shared Portal items like a Git remote.
gitmap push origin main
Features

Everything you'd expect from Git,
built for GIS.

Works with ArcGIS Online and Enterprise Portal. Python 3.11–3.13. 450+ tests. MIT licensed.

🌿
Branching
Create isolated branches to safely develop and test map changes without touching production.
📝
Commits & History
Save named snapshots with messages. Full log with author, date, and what changed.
🔍
Visual Diff
See exactly what changed between any two commits or branches — layers, opacity, visibility, symbology.
🔀
Merge
Bring branches back together. Conflict detection keeps your maps safe.
⏪
Revert & Reset
One command to undo any commit or restore a historical map state.
🔁
Push & Pull
Sync map state to/from ArcGIS Portal — share across teams and environments.
⚙️
ArcGIS Pro Toolbox
9 native geoprocessing tools so you can run GitMap workflows directly inside ArcGIS Pro.
🤖
Auto-Pull
Schedule automatic syncs to keep local repos in step with live Portal maps.
Get Started

Up and running in 2 minutes.

Requires Python 3.11+ and an ArcGIS Online or Portal account.

# Install from PyPI
pip install gitmap

# Initialize a repo for an existing web map
gitmap init --portal https://org.maps.arcgis.com --item-id <map-id>

# Commit the current state
gitmap commit -m "Initial commit"

# Create a branch and start experimenting
gitmap branch feature/my-changes && gitmap checkout feature/my-changes
